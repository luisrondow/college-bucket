# AF 2.2

## 1. Erro Identificado

O ficheiro `arquivo.h` utiliza `#endif` sem a correspondente diretiva `#ifndef` (ou `#if`/`#ifdef`) no início. Ou seja, falta o header guard completo. O código original é:

```c
/* arquivo.h */

int calcular(int x);

#endif
```

O `#endif` está órfão. Não existe nenhuma diretiva condicional de abertura que lhe corresponda.

## 2. Código Corrigido

```c
/* arquivo.h */
#ifndef ARQUIVO_H
#define ARQUIVO_H

int calcular(int x);

#endif /* ARQUIVO_H */
```

## 3. Explicação

O pré-processador do C utiliza diretivas condicionais (`#ifndef`/`#define`/`#endif`) como header guards para impedir a inclusão múltipla de um mesmo ficheiro. Sem o `#ifndef ARQUIVO_H` e o `#define ARQUIVO_H`, o `#endif` isolado não tem par de abertura, o que gera um erro de compilação (tipicamente `error: #endif without #if`). Além disso, mesmo que o `#endif` fosse simplesmente removido, o ficheiro ficaria desprotegido: se outro módulo o incluísse mais do que uma vez (direta ou indiretamente), a declaração `int calcular(int x);` seria repetida. Embora declarações repetidas de funções sejam toleradas em C, a ausência de guards torna-se problemática quando o header contém definições de tipos, variáveis ou macros, levando a erros de redefinição. A versão corrigida garante que o conteúdo do header é processado apenas uma vez por unidade de tradução.
