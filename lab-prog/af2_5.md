# AF 2.5

## Código com erro

**modulo_suspeito.h**
```c
/* modulo_suspeito.h */
#define MAX 100
int buffer[MAX];  // Será que isto está correto?
void processar();
```

**modulo_suspeito.c**
```c
/* modulo_suspeito.c */
#include <stdio.h>
// Falta alguma coisa aqui?
static int privada;
int publica;
void helper() {
    printf("Ajudante\n");
}
void processar() {
    helper();
}
```

## Problemas Identificados

### Problema 1: Ficheiro `.h` sem *include guards*

- **Porquê:** Esquecer o header guard (ou include guard) num ficheiro de cabeçalho é um erro comum que resulta, principalmente, em erros de compilação devido a múltiplas redefinições de símbolos. Sem esse mecanismo, o pré-processador não consegue garantir que o conteúdo do ficheiro `.h` seja processado apenas uma única vez.
- **Correção:** Adicionar `#ifndef MODULO_SUSPEITO_H` e `#define MODULO_SUSPEITO_H` no início do ficheiro e `#endif /* MODULO_SUSPEITO_H */` no final.

### Problema 2: Função `void helper()` implementada no ficheiro `.c` porém não declarada no ficheiro header

- **Porquê:** Se `helper()` se destina a ser utilizada apenas internamente pelo módulo, deve ser marcada como `static` para restringir a sua ligação (linkage) ao ficheiro onde é definida. Caso contrário, o símbolo fica visível globalmente durante a fase de linkagem, podendo colidir com funções homónimas noutros módulos. Se, por outro lado, se pretende que seja pública, a sua declaração deve constar no ficheiro `.h` para que outros módulos a possam invocar de forma segura, com verificação de tipos pelo compilador.
- **Correção:** Adicionar `static` antes da implementação no ficheiro `.c` (caso seja de uso interno), ou adicionar `void helper();` no ficheiro `.h` (caso se pretenda que seja pública).

### Problema 3: Variável `int buffer[MAX]` declarada no ficheiro header sem a keyword `extern`

- **Porquê:** Quando uma variável é declarada diretamente num ficheiro `.h` sem `extern`, cada unidade de tradução (ficheiro `.c`) que inclua esse header cria a sua própria definição da variável. Isto resulta num erro de linkagem por múltipla definição (*multiple definition*), pois o linker encontra vários símbolos com o mesmo nome. A keyword `extern` indica ao compilador que a variável é apenas declarada (não definida) naquele ponto, devendo a sua definição existir num único ficheiro `.c`.
- **Correção:** No ficheiro `.h`, declarar como `extern int buffer[MAX];`. No ficheiro `.c`, definir a variável com `int buffer[MAX];`.

### Problema 4: Funções e variáveis com nomes genéricos que podem causar colisões de nomes

- **Porquê:** Nomes como `buffer`, `processar` e `helper` são demasiado genéricos. Em projetos com múltiplos módulos, é muito provável que outro módulo utilize nomes idênticos, resultando em conflitos de ligação (*linker conflicts*). A utilização de um prefixo associado ao módulo reduz significativamente este risco.
- **Correção:** Utilizar um prefixo que identifique o módulo:
```c
/* modulo_suspeito.h */
extern int Modulo_buffer[MAX];
void Modulo_processar();
```

### Problema 5: Falta a definição da variável `buffer` no ficheiro `.c`

- **Porquê:** Ao corrigir o Problema 3, a declaração no header passa a ser apenas `extern`, ou seja, uma referência externa. É necessário que exista uma (e apenas uma) definição real da variável numa unidade de tradução. Sem essa definição no ficheiro `.c`, o linker emitirá um erro de símbolo indefinido (*undefined reference*).
- **Correção:** Adicionar `int buffer[MAX];` (ou `int Modulo_buffer[MAX];`, caso se aplique o prefixo do Problema 4) no ficheiro `modulo_suspeito.c`.

### Problema 6: Falta a declaração da variável pública `int publica` no ficheiro `.h`

- **Porquê:** A variável `publica` é definida no ficheiro `.c` sem a keyword `static`, o que a torna visível globalmente. No entanto, sem uma declaração `extern` correspondente no ficheiro `.h`, outros módulos que pretendam utilizá-la não têm forma de a referenciar de maneira segura. Declarar a variável no header permite que o compilador verifique a consistência de tipos entre módulos.
- **Correção:** Adicionar `extern int publica;` no ficheiro `modulo_suspeito.h`.

## Código Corrigido

**modulo_suspeito.h**
```c
/* modulo_suspeito.h */
#ifndef MODULO_SUSPEITO_H
#define MODULO_SUSPEITO_H

#define MAX 100

extern int buffer[MAX];
extern int publica;

void processar();

#endif /* MODULO_SUSPEITO_H */
```

**modulo_suspeito.c**
```c
/* modulo_suspeito.c */
#include <stdio.h>
#include "modulo_suspeito.h"

int buffer[MAX];
static int privada;
int publica;

static void helper() {
    printf("Ajudante\n");
}

void processar() {
    helper();
}
```
